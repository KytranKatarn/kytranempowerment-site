Tips & Tricks

Hard-won lessons from building A.R.C.H.I.E. — a 129-agent, 16-department AI platform.

🧠

Claude Code Tips

Maximize your productivity with Claude Code's project memory and context system.

CLAUDE.md Is Your Project Memory

Put architecture decisions, patterns, and conventions in CLAUDE.md. Claude reads it every session. Think of it as your project's brain — the more precise it is, the better Claude performs.

Split Rules by Domain

Use .claude/rules/ for domain-specific knowledge. We have database.md, frontend.md, ai-bridge.md, starbase.md, and more. Each file loads focused context for its area.

Search Before Building

With 18,000+ knowledge base entries, we never reinvent the wheel. Always search what exists first. Past decisions, mistakes, and patterns are all indexed and searchable.

Plan Mode for 3+ Steps

Break complex work into phases. Get alignment before coding. Our rule: if a task touches 3+ files or involves architectural decisions, enter plan mode first.

Two Strikes and Ask

If two attempts fail, stop and ask. Don't spiral through 20 tries. This single rule has saved us more time than any other. Know when to escalate.

Verify Everything

Never claim done without live proof. Our workflow: Edit, Commit, Deploy, Test, then Claim. Screenshots or it didn't happen. Anti-rationalization is a real pattern.

.md File Conventions

Structure your project knowledge for maximum AI comprehension and cross-session persistence.

CLAUDE.md — Project Level

Architecture overview, key patterns, file locations, common errors, and deployment commands. This is the single source of truth for how your project works.

.claude/rules/*.md — Domain Rules

Loaded per-session based on context. Database gotchas, frontend patterns, API conventions, deployment rules. Each file is a specialist's handbook.

Memory Files — Cross-Session

User preferences, project state, behavioral feedback, reference data. These persist across conversations and keep Claude aligned with your working style.

Spec Docs — Before Building

Design specs before implementation. Reviewed and approved before any code is written. Saves rework and ensures alignment on requirements.

Decision Logs — ADRs

Log every architectural decision with context. Six months later, you'll know why you chose PostgreSQL over SQLite, or why agents are 1-per-station.

Mistake Logs — Learn Once

Store mistakes with knowledge_type='mistake_learned'. These get 1.3x search weight. The system actively prevents repeating costly errors.

Agent Coordination Patterns

Patterns for orchestrating multi-agent AI systems at scale.

Central Dispatch, Always

All LLM calls route through a central Bridge dispatcher. Never call models directly from modules. This gives you logging, routing, fallback, and governance in one place.

Department-Based Routing

Organize agents by business function, not abstract categories. Analytics, Creative, Engineering, Security — each department has specialists with relevant skills.

Warm Boot Priority

Keep critical models loaded in GPU/RAM. Share LLMs across agents in the same department. Cold-loading a 55GB model takes 9 minutes — warm boot is instant.

Escalation Chains

Local department → same floor → cross-department → fleet node → cloud fallback. Each level adds latency but widens capability. Most tasks resolve at tier 1.

Agent Welfare Matters

Track agent workload, rest cycles, and stress. It sounds whimsical, but it enforces real constraints: max 3 concurrent jobs, mandatory cooldowns, shift rotations.

Approval Tiers

Tier 1 (auto): notifications, KB updates, internal logs. Tier 2 (human): cloud escalation, agent creation, cost-sensitive ops. Autonomous doesn't mean uncontrolled.

Operations & Infrastructure

Production lessons from running a self-hosted AI platform 24/7.

Self-Host Everything

Full data sovereignty. No vendor lock-in. Our entire platform runs on a single server with 128GB RAM and a GPU. Cloud is a fallback, not a dependency.

Docker for Isolation

Platform, database, and ComfyUI each run in separate containers. One restart doesn't take down everything. Compose files make the whole stack reproducible.

Model Tiers: GPU > CPU > Cold

Three-tier model management. GPU for fast inference, CPU for capacity, cold storage on SSD. Auto-promote on demand, auto-demote on idle. VRAM is precious.

Background Jobs, Not Blocking

57 scheduled background jobs handle monitoring, scanning, training, and maintenance. Users never wait. Return a job ID immediately, process asynchronously.

Idempotent Migrations

Every database migration uses IF NOT EXISTS and ON CONFLICT DO UPDATE. Run them twice, ten times — same result. No fear of re-running.

Monitor Model VRAM, Not Just Disk

A 2GB model on disk can use 5.5GB in VRAM with context. Always account for num_ctx inflation. Runtime size is roughly 1.3x disk size.
